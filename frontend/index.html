🔧 Trigger Incident

This simulates a database misconfiguration in the Order Service by calling it with an invalid connection. In a real scenario, you would run the following docker command:

# Simulate: Change the DB hostname to an invalid value
docker compose stop order-service
docker compose run -e DATABASE_URL=postgresql://admin:password@wrong-host:5432/microservices order-service

🧪 Root Cause Analysis

Root Cause: DATABASE_URL environment variable misconfigured — hostname set to "wrong-host" instead of "postgres"

Log Analysis Commands:

# Check container logs
docker logs order-service --tail 50

# Expected error output:
ERROR: could not connect to server: Name or service not known
Is the server running on host "wrong-host" and accepting TCP/IP connections on port 5432?

# Check container status
docker compose ps

# Inspect environment variables
docker inspect order-service | grep DATABASE_URL

Contributing Factors:

  • No pre-deployment config validation
  • Missing DB connectivity health checks at startup
  • No automated rollback on startup failure
  • Alert threshold too slow (2 min detection time)

🛠️ Mitigation & Resolution

Resolution Steps: Fix the DATABASE_URL and restart the Order Service
# Step 1: Stop the broken container
docker compose stop order-service

# Step 2: Fix environment variable in docker-compose.yml
# Change DATABASE_URL back to: postgresql://admin:password@postgres:5432/microservices

# Step 3: Restart service with correct config
docker compose up -d order-service

# Step 4: Verify service is healthy
curl http://localhost/orders-svc/health

# Step 5: Confirm orders are working
curl -X POST http://localhost/orders-svc/orders \
  -H "Content-Type: application/json" \
  -d '{"user_id": 1, "product_id": 1, "quantity": 2}'
